Both of these properties are used to get the width of an element on the page, but there are a few differences:
  • Return values:
    • getComputedStyle(elem).width - Returns the width of an element, including borders, margins, and padding. The value is returned in pixels or another CSS-defined unit (such as %, em, etc.).
    • elem.clientWidth - Returns the width of an element, excluding borders, margins, and scrolling. The value is returned in pixels.
  • Ability to set value:
    • getComputedStyle(elem).width - cannot be used to set an element's width value. This is read-only.
    • elem.clientWidth - also cannot be used to set an element's width value. This is read-only.
  • Browser support:
    • getComputedStyle(elem).width - supported by all modern browsers.
    • elem.clientWidth - Also supported by all modern browsers, but in IE8 and earlier, this method does not work correctly in standards-compliant mode.
  • Calculation of values:
    • getComputedStyle(elem).width - Used to get the actual width of an element after all styles have been applied, including values inherited from the parent element.
    • elem.clientWidth - used to get the width of an element, excluding borders, margins, and scrolling. If the element has a scrollbar, then the clientWidth value will be less than the element's width.
